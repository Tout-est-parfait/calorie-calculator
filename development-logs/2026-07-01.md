# 开发日志 — 2026-07-01

## 今日完成

### Phase 15：跨设备登录修复 + 字段映射 + UI 修复

- [x] **跨设备登录修复**：
  - `js/auth.js`：`loginUser()` 改为发送明文密码到服务端（HTTPS 加密）
  - `functions/api/auth.js`：`handleLogin()` 新增 `serverHash()` 函数，服务端用存储的盐做 SHA-256 哈希后与数据库比对
  - `js/db.js`：`loginRemote()` 发送 `password` 字段（明文）而非 `passwordHash`
  - 优点：新设备无需知道注册时的盐值，服务端统一验证

- [x] **字段名规范化（修复 undefined 显示）**：
  - `js/db.js`：新增 `normalizeRecord()` 函数，将 D1 返回的 snake_case 字段转为客户端 camelCase
  - 应用于 `getRecordsDB()` 和 `syncFromServer()` 两个数据路径
  - 修复了食物名称、份量标签等显示 "undefined" 的问题

- [x] **搜索栏 UI 重叠修复**：
  - `css/style.css`：`.search-add-custom` 添加 `flex-wrap: wrap`，窄屏设备上文字不再重叠

### Phase 16：功能改进 — 强制登录 + AI 本地化 + 日期可见性

- [x] **移除匿名登录**：
  - `js/app.js`：删除 `onAuthSkip()` 函数和 `auth-skip` 事件绑定
  - `index.html`：删除 auth-skip 按钮，替换为静态文字「请登录或注册以使用热量计算器」
  - 所有用户必须先注册/登录才能使用

- [x] **AI 数据仅本地存储**：
  - `js/api.js`：`saveAIAdvice()` / `loadAIAdvice()` / `saveMealPlan()` / `loadMealPlan()` 改为同步函数，仅操作 localStorage
  - `js/db.js`：`syncFromServer()` 和 `migrateToServer()` 移除 AI 缓存同步
  - `js/advisor.js`：移除对上述函数的 `await` 调用
  - AI 建议和食谱数据不会上传到 D1 云端

- [x] **AI 功能按日期可见性**：
  - `js/app.js`：`showDashboardView()` / `switchTab()` / `refreshView()` 增加日期判断逻辑
    - AI 建议区域：仅今日可见
    - AI食谱 Tab：今日及未来可见，过往日期隐藏并自动切回仪表盘
  - `js/advisor.js`：`renderAdvice()` 中 AI 分析按钮仅当日显示
  - `generateMealPlan()`：当日摄入记录传给 AI 作为下一餐建议的上下文
  - `js/api.js`：`getDailyMealPlan()` 新增 `todayIntakeContext` 参数
  - `js/advisor.js`：`renderMealPlanResultHTML()` 新增下一餐建议卡片样式

### Bug Fix：advisor.js 重复 const 声明

- [x] **问题**：`js/advisor.js` — `generateMealPlan()` 函数内 `const dateStr` 在第 578 行和第 638 行重复声明
- [x] **影响**：`SyntaxError: Identifier 'dateStr' has already been declared` → advisor.js 解析失败 → `renderAdvice` 等函数未定义 → `initApp()` 在 `await renderAdvice()` 处抛出 ReferenceError → `bindEvents()` 未执行 → **所有按钮点击无反应**
- [x] **修复**：删除第 638 行重复的 `const dateStr`，复用第 578 行的变量（值相同）
