# 项目需求文档

## 项目概述

**项目名称**：网页版热量计算器

**项目目标**：为用户提供每日食物热量记录与管理工具，根据科学标准分析摄入情况，提供饮食与运动建议。

**目标用户**：支持多用户登录（可选），也支持匿名使用。

## 用户模式

- 支持多用户登录（用户名 + 密码），每个用户数据独立隔离
- 也支持匿名使用（跳过登录，数据存储在浏览器本地）
- 数据存储在浏览器本地（localStorage）
- 打开网页即用，无需后端服务

## 核心功能

### 1. 食物搜索与录入

- 内置常见食物热量数据库（涵盖中餐常见食材、菜品、零食饮料）
- 用户通过搜索框输入食物名称，实时显示匹配结果
- 点击搜索结果，选择份量（如 100g、1 碗、1 个），自动计算热量
- 支持添加到「今日摄入列表」
- 数据库中没有的食物，允许用户自定义添加名称和热量

### 2. 每日热量仪表盘

- 显示今日已摄入总热量（大卡 / kcal）
- 显示三大营养素：碳水、蛋白质、脂肪
- 可设置目标热量线（默认参考中国居民膳食指南）
- 进度条直观展示当日摄入占目标比例

### 3. 科学建议引擎

系统根据当日摄入数据，自动给出建议：

| 场景 | 建议内容 |
|------|----------|
| 热量超标 | 超标多少 kcal → 建议运动类型及时间 |
| 热量不足 | 建议补充高营养密度食物 |
| 碳水偏高 | 提醒减少精制碳水，推荐替代食物 |
| 蛋白质不足 | 推荐高蛋白食物 |
| 脂肪偏高 | 提醒注意油脂摄入，推荐清淡饮食 |

### 4. 历史记录

- 按日期查看过往饮食记录
- 简单统计：最近 7 天平均摄入、最高/最低日

### 5. 个性化设置

- 体重管理目标：减重 / 保持 / 增重
- 根据目标自动调整每日推荐热量
- 自定义热量目标

### 6. 运动消耗参考

- 内置常见运动的热量消耗参考数据
- 根据超标热量，推荐具体运动方案

### 7. 用户登录系统

- 用户名 + 密码注册和登录
- 密码安全哈希存储（Web Crypto API）
- 每个用户独立数据空间，互不干扰
- 支持匿名使用（跳过登录）
- 登出/切换用户

### 8. AI 智能功能（DeepSeek API）

- **AI 食物估计**：输入自然语言描述，AI 自动估计热量和营养素
- **AI 智能建议**：发送每日饮食数据，获取个性化饮食与运动建议
- **AI 建议持久化**：AI 分析结果自动保存，退出后重进无需重新请求
- 支持代理模式（推荐），无需填写 API Key 即可使用全部 AI 功能
- 代理模式内置默认密钥（服务端持有，用户不可见）
- 高级用户可填写自己的 API Key（直接模式或代理模式均支持）
- 规则建议引擎保留，AI 作为增强功能

### 9. AI 每日食谱计划

- 底部导航独立 Tab：「AI食谱」
- AI 根据用户体重目标（减重/保持/增重）和每日热量目标生成全天食谱
- 食谱包含：早餐、午餐、晚餐、加餐（每餐含食物、热量、说明）
- 配套训练计划：运动类型、动作列表、时长
- 日常饮食小贴士
- 食谱持久化存储，按日期隔离，切换日期不混淆

## 界面导航

底部 3Tab 导航：
- **仪表盘**：膳食记录、营养仪表盘、搜索、AI建议
- **历史记录**：7天统计、历史日期列表
- **AI食谱**：每日 AI 定制食谱与训练计划

每个子界面顶部提供「← 返回」按钮，方便手机单手操作。

## 界面语言

全中文界面。

## 平台支持

- 桌面浏览器
- 移动端浏览器（移动端优先设计）
- PWA 支持（添加到手机主屏幕，离线可用）
- Cloudflare Pages 部署（含 API 代理 Functions）
