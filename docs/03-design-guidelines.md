# 设计规范

## 设计理念

**清新、简洁、可信赖**。面向健康管理场景，配色柔和自然，信息层次清晰，操作路径短。

## 色彩系统

| 角色 | 色值 | 用途 |
|------|------|------|
| 主色（健康绿） | `#4CAF50` | 主按钮、进度条、强调元素 |
| 主色浅 | `#E8F5E9` | 卡片背景、选中态 |
| 警告色 | `#FF9800` | 热量超标警告 |
| 危险色 | `#F44336` | 严重超标、删除按钮 |
| 信息色 | `#2196F3` | 建议卡片、链接 |
| 文字主色 | `#212121` | 标题、正文 |
| 文字辅色 | `#757575` | 说明文字、标签 |
| 背景色 | `#FAFAFA` | 页面背景 |
| 卡片白 | `#FFFFFF` | 卡片、列表项背景 |

## 字体

```css
--font-family: -apple-system, BlinkMacSystemFont, "Segoe UI",
              "PingFang SC", "Microsoft YaHei", sans-serif;
```

- 标题：字重 600，字号 18-24px
- 正文：字重 400，字号 14-16px
- 辅助文字：字重 400，字号 12-13px

## 圆角与间距

- 卡片圆角：12px
- 按钮圆角：8px
- 输入框圆角：8px
- 页面内边距：16px（移动端）/ 24px（桌面端）
- 卡片内边距：16px
- 元素间距：8px（紧凑）、12px（常规）、16px（宽松）

## 阴影

- 卡片阴影：`0 2px 8px rgba(0,0,0,0.08)`
- 悬浮阴影：`0 4px 16px rgba(0,0,0,0.12)`

## 布局原则

### 移动端优先

- 最大宽度 480px，居中显示
- 桌面端适当增加两侧留白
- 使用 Flexbox + Grid 布局

### 页面分区（从上到下）

1. **顶部导航栏** — 日期选择器、设置入口
2. **热量仪表盘** — 环形进度条，显示今日摄入/目标
3. **三大营养素卡片** — 碳水、蛋白质、脂肪，横向排列
4. **搜索栏** — 食物搜索与添加入口
5. **今日摄入列表** — 已添加的食物，可删除
6. **科学建议卡片** — 根据数据动态生成
7. **底部导航** — 仪表盘 / 历史记录

## 交互规范

- 搜索：输入即搜，防抖 300ms
- 添加食物：点击搜索结果 → 选择份量 → 确认添加（弹窗）
- 删除记录：左滑或点击删除图标 → 确认
- 切换日期：顶部左右箭头切换
- 所有数据操作即时保存到 localStorage

## 响应式断点

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| 移动端 | < 600px | 单列，全宽卡片 |
| 桌面端 | ≥ 600px | 最大宽度 480px 居中，留白增加 |

## 空状态设计

- 无记录时显示友好插图（emoji）+ 引导文字
- 搜索无结果时显示"未找到，试试自定义添加"
